# Decision Log

## 2026-07-11

### Decision
Separate HTML, CSS, and JavaScript into individual files.

**Reason**
Improve maintainability and prepare for future modularization.

---

### Decision
Use GitHub as the central location for project documentation.

**Reason**
Keeps documentation version-controlled alongside the source code.

---

### Decision
Build the app as a Financial Operating System rather than a budgeting app.

**Reason**
The focus is helping users make better financial decisions, not simply tracking spending.

---

### Decision
Develop the app using plain HTML, CSS, and JavaScript.

**Reason**
Keeps the application lightweight, easy to maintain, and compatible with GitHub Pages.

---

### Decision
Build the product mobile-first.

**Reason**
The primary experience will be on mobile devices and eventually distributed through the Google Play Store.
