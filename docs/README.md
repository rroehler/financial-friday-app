# Documentation Philosophy

Financial Friday documentation is organized by **purpose**, not by file type.

Every document should answer one of four questions:

- **What are we building?** → Product
- **How are we building it?** → Architecture
- **What are we building next?** → Planning
- **What have we already built?** → Release Notes

Meeting notes provide historical context for decisions but are not considered permanent documentation.

---

# Keeping Documentation Current

Documentation is considered part of every feature.

When functionality changes, the appropriate documentation should be reviewed and updated before the feature is considered complete.

Documentation updates are included as part of the project's release workflow.
